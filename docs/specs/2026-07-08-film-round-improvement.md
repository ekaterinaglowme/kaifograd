# Раунд «Угадай фильм» — спека на улучшение (на согласование)

Дата: 2026-07-08. Статус: **черновик на согласование** — не реализуем, пока Катя не подтвердит.

## Как раунд работает сейчас

- 7 вопросов. На каждом показывается **заблюренный кадр** (`film-N-blur.png`) и поле ввода.
  Команды вписывают название фильма. Таймер 45 сек.
- После 7-го вопроса статус `round_review`: ведущая листает **7 слайдов с оригинальным
  кадром** (`film-N-original.png`) и правильным названием.
- Затем — отсчёт 3-2-1 и итоги раунда (+ общий счёт).

## Как программа сейчас засчитывает ответ

1. Ответ приводится к «нормальному виду»: нижний регистр, `ё→е`, убираются знаки
   препинания и лишние пробелы.
2. Засчитывается, если нормализованный ответ **точно совпадает** с одним из принимаемых
   вариантов, **или содержит** его (для вариантов длиннее 4 символов).
3. Принимаемые варианты (`acceptedAnswers`) заданы вручную. Пример: «Бойцовский клуб»
   принимает `бойцовский клуб`, `бойцоввский клуб`.

**Минус:** опечатка на 1 букву, которой нет в списке → не засчитано. Все опечатки заранее
не пропишешь.

## Проблемы, которые ты назвала

1. Непонятно, **кто что сейчас отвечает**.
2. Непонятно, **как показываются ответы**.
3. Непонятно/жёстко, **как засчитывается угадывание**.

## Предложение

### 1. Нечёткое сравнение по расстоянию Левенштейна (прощаем опечатки)

**Расстояние Левенштейна** — минимальное число правок (вставить / удалить / заменить
одну букву), чтобы превратить один текст в другой. Примеры:

- `матрица` → `матриса` = **1** (заменили ц→с)
- `интерстеллар` → `интерстелар` = **1** (пропущена «л»)
- `титаник` → `корабль` = много (совсем другое слово, не засчитаем)

**Как применим:** ответ засчитывается, если он проходит по текущим правилам **или** его
расстояние Левенштейна до любого принимаемого варианта ≤ порога. Порог зависит от длины
слова (чтобы короткие слова не «прощались» слишком легко):

| Длина принимаемого варианта | Разрешённых опечаток |
|-----------------------------|----------------------|
| ≤ 5 букв                    | 1                    |
| 6–10 букв                   | 2                    |
| больше 10 букв              | 3                    |

Это прощает 1–3 опечатки, но не засчитывает совершенно другой ответ. Работает для всех
текстовых раундов (кино, «Где я», «Баг», «Культурный сервер», мемы) — не только кино.

### 2. Показывать, кто что ответил и кто угадал

- **У ведущей** в кино-раунде — список команд с их ответами и пометкой ✓/✗ по ходу
  (как в обычных раундах). Проверим, что видно и корректно.
- **На экране раскрытия** (`round_review`), для каждого фильма показывать: оригинальный
  кадр + правильное название + **список команд, которые угадали** (с ✓). Так всем ясно,
  кто отгадал этот кадр.

### 3. Прозрачные подписи (только UI, логику не трогаем)

- **Команда:** «Вопрос N/7 · впиши название фильма» + кадр + поле. После отправки —
  «Ответ принят».
- **Наблюдающий (во время вопроса):** кадр + «Вопрос N/7» + таймер, **без ответов**
  (чтобы не спойлить).
- **Раскрытие:** оригинал + правильный ответ + кто угадал.
