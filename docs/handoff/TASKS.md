# Кайфоград — что ещё доделать (задачи для продолжения)

Приоритеты: **P0** — без этого игра на мероприятии не состоится; **P1** — важно к игре;
**P2** — полезно/подстраховка. Каждая задача самодостаточна: путь к файлам + контекст.

---

## P0 — деплой: одна ссылка для всех

**Цель:** приложение доступно по одной публичной ссылке `https://домен/`, команды
заходят со своих компьютеров, ведущая — вкладка «Ведущая» + код `0306`, проектор —
`?view=screen`.

Что сделать:
1. Развернуть Node-сервер (`server.js`) на публичном хосте (например, VM как у PetBank
   `212.147.238.3`, либо любой PaaS с Node 18+). Запуск: `node server.js`.
2. Переменные окружения при старте: `PORT` (или проксировать 80/443 → порт),
   `HOST_CODE=0306` (можно сменить), `TEAMS=<число команд>`.
3. Реверс-прокси + HTTPS. **Важно для SSE**: `proxy_buffering off;`,
   `proxy_read_timeout 1h;`, не сжимать `text/event-stream`. Иначе realtime не идёт.
4. Автоперезапуск (systemd/pm2) и запись `game-state.json` в рабочей папке (нужны
   права на запись).
5. Проверить с 2+ реальных устройств (см. P1 «многоустройственный тест»).

Файлы: `server.js`. Ничего в коде менять не обязательно — всё берётся из env.

## P1 — картинки мемов и эстафеты

**Цель:** в раунде 6 «Озвучь мем» под каждым вопросом своя картинка; в раунде 8
«Кайфуй и работай» — картинка эстафеты. Сейчас там аккуратные заглушки.

Что сделать:
1. Положить файлы в `assets/` (например, `assets/mem-1.jpg … mem-7.jpg`,
   `assets/estafeta.jpg`).
2. В `src/rounds.js` проставить путь в поле `image` каждого вопроса раунда «Озвучь мем»
   (сейчас `image: ""`) и в объекте раунда «Кайфуй и работай» (`image: ""`).
3. Клиент уже умеет: если `image` пуст — показывает заглушку, если задан — картинку.
   Проверить, что файл реально отдаётся (расширения .jpg/.png/.webp сервер знает).

Соответствие мемов порядку вопросов — см. `src/rounds.js` (раунд «Озвучь мем»):
1) Чипи чипи чапа чапа 2) Я выживу… 3) Кандибобер 4) Без пяти минут пятого утра
5) Наталья, морская пехота 6) Вы кто такие 7) ИванГай, хаю хай.

## P1 — многоустройственный тест после деплоя

Пройти реальный сценарий с 2–3 устройств:
1. Две команды заходят по общей ссылке, каждая берёт свой слот, имя, цвет, получает
   токен; проверить, что открыть занятую команду с другого устройства нельзя.
2. Ведущая (код `0306`) запускает раунд; команды отвечают; проверить, что чужие ответы
   не видны, таймер и автопереход работают, проектор показывает вопрос и рейтинг.
3. Аварийные кнопки: закрыть вопрос, пересчитать, поправить баллы, сброс (с
   подтверждением).
4. Раунд 7 (песня, 2 поля) и раунд 8 (мячи, ручные баллы, «Завершить раунд»).
5. Финал 3-2-1 → топ-3.
6. Проверить переподключение SSE (обновить страницу/потерять сеть на секунду).

Локально API проверялся так (для справки): reset → join team1/team2 → занятый
цвет/номер отклоняются → startRound → submitAnswer → приватность (команда видит
только свой ответ, проектор — ничего) → scoreNow (+баллы) → adjustScore → защита
кодом → SSE отдаёт data → finalReveal → персист в файл.

## Обновление 2026-07-07 (сделано в этой сессии)

- **Ведущая контролирует конец раунда.** После последнего вопроса сервер ставит
  статус `round_over` (не авто-итоги); показывается «Раунд закончился», ведущая
  жмёт «Завершить раунд» → итоги. Проверено сквозным API-прогоном + headless-рендером
  (host/команда/проектор показывают «Раунд закончился»). Файлы: `server.js` (тик +
  действие `finishRound`), `src/app.js` (ветки `round_over`, кнопка `finish-round`).
- **Miro больше не лагает.** Причина: клиент перерисовывал весь DOM ~2 раза/сек во
  время раунда, из-за чего Miro-iframe пересоздавался. Исправлено: таймерная
  перерисовка пропускает вкладку карты; `render()` не пересобирает DOM, пока открыта
  загруженная карта; сам iframe грузится только по кнопке «Показать карту здесь»
  (по умолчанию — лёгкая заглушка + «Открыть в Miro»). Файлы: `src/app.js`
  (`renderCity`, `render`, интервал, действие `load-miro`, `state.miroLoaded`).
