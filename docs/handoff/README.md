Дата среза: 2026-07-07.

## Что это

Командный квиз «Кайфоград» для мероприятия в институте. Одна общая ссылка: команды
заходят со своих компьютеров, ведущая управляет по коду, проектор показывает вопрос
и конкурсные таймеры. Всё синхронизируется в реальном времени через сервер.

## Архитектура (коротко)

- `server.js` — Node http-сервер. Держит **одно** состояние игры в памяти, раздаёт
  статику и API. Серверный «тик» ведёт таймер вопросов, автопереходы и финальный
  отсчёт. Состояние сохраняется в `game-state.json` (переживает перезапуск).
- `src/game.js` — чистая игровая логика (создание игры, старт раунда, регистрация
  команды, приём ответа, подсчёт, пересчёт, ручная правка баллов, итоги, финал,
  `serializeForViewer` для приватности). Импортируется и сервером, и тестами.
- `src/rounds.js` — единый источник данных 8 раундов (вопросы/ответы) и описаний
  ресурсов города. Импортируется сервером и клиентом.
- `src/app.js` — клиент (браузер). Рендер экранов Ведущая/Команда/Проектор/Miro.
  Получает состояние по SSE (`/api/events`), действия шлёт `POST /api/action`,
  начальное состояние — `GET /api/state`. Личность команды хранится токеном в
  `sessionStorage`.
- `index.html`, `styles.css` — оболочка и стили. Лого-кот: `assets/kaifograd-cat.jpg`.
- `tests/game.test.js` — юнит-тесты логики (`node --test`).

### API сервера

- `GET /api/state?view=&team=&token=&code=` — текущее состояние (с редакцией под роль).
- `GET /api/events?view=&team=&token=&code=` — SSE-поток: новое состояние на каждое
  изменение. Пинг-комментарии каждые 25с.
- `POST /api/action` — тело JSON `{ type, ...payload }`.
  - Команда: `join` (возвращает `token`), `submitAnswer` (нужен `token`).
  - Ведущая (нужен `code`): `startRound`, `scoreNow`, `nextQuestion`,
    `togglePause`, `recount`, `adjustScore`, `finishRound`, `nextRound`,
    `startManualAttempt`, `toggleManualAttemptPause`, `finishManualAttempt`,
    `closeManualRound`, `finalReveal`, `reset`.

### Приватность

- Команда видит только свой ответ на текущий вопрос; чужие ответы и `token` не
  отдаются. Ответ принимается только со своим токеном (чужое устройство/вкладка на
  занятой команде → «Команда уже занята»).
- Ведущая (с кодом) видит все ответы. Проектор — не видит ответов.

## Игра: 8 раундов (по порядку)

1. Разминка — микс тем (4 варианта)
2. Цена вопроса — «что дороже» (4 варианта)
3. Где я, если это не Google Maps? (текст)
4. Баг в истории (текст + 1 вариант)
5. Культурный сервер (текст + 1 вариант)
6. Озвучь мем — картинка + ввод ответа (нужны картинки)
7. Угадай песню — 2 поля: исполнитель + название, +1/+1
8. Кайфуй и работай — эстафета с мячами, баллы вручную (финал, нужна картинка)

## Как запускать локально

```
cd /Users/ekaterina/kaifograd-prototype
/Users/ekaterina/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node server.js
```

- Команды: `http://localhost:4173/`
- Ведущая: `http://localhost:4173/?view=host`, код `0306`
- Проектор: `http://localhost:4173/?view=screen`

Переменные окружения: `PORT` (по умолчанию 4173), `HOST_CODE` (по умолчанию `0306`),
`TEAMS` (число слотов команд, по умолчанию 6).

## Как проверять

- Тесты логики: `node --test tests/*.test.js` (должны проходить).
- Синтаксис: `node --check src/app.js && node --check server.js`.
- API целиком (без браузера): см. пример прогона в TASKS.md (join → token →
  приватность → подсчёт → reset). Есть готовый скрипт-пример в истории (api-check).

## Уже сделано

- Контент 8 раундов, автоподсчёт всех типов (варианты/число/текст/песня-2-поля),
  ручной финал с мячами.
- Экран ведущей — полноценный пульт: начать игру, закрыть вопрос, перейти к
  следующему вопросу, поставить паузу, завершить раунд, пересчитать, поправить
  баллы (± по командам), запустить 60-секундные попытки в раунде 8, сбросить игру
  + **подтверждения** на сброс и финал.
- Backend: серверное состояние, SSE realtime, POST-действия, персист в файл,
  приватность через токены, код ведущей, серверный таймер и финал 3-2-1.
- Клиент работает через сервер (SSE + POST), токен команды хранится в
  sessionStorage, занятые цвет/номер недоступны, интерфейс использует термин
  «команда».
- Вкладки: Ведущая / Команда / Miro / Проектор (проектор без админки).
- Тесты проходят; API и базовый браузерный сценарий проверены.

## Открытые решения (нужно слово Кати)

- **Таймеры зафиксированы тестом**: раунд 1 — 20с, раунды 2–6 — 45с, раунд 7 —
  30с, раунд 8 — 60с на повторяемую попытку.
- **Число команд** на игре: задать `TEAMS` (сейчас 6). Если команд может быть больше —
  поднять значение.
- **Картинки**: 7 мемов (раунд 6) + эстафета (раунд 8) — Катя пришлёт; вставить в
  `assets/` и прописать `image` в `src/rounds.js`.

## Важные детали/риски для деплоя

- Перед мероприятием нажать «Сбросить игру» или удалить `game-state.json` (там
  сохранится тестовое состояние).
- SSE за реверс-прокси: отключить буферизацию (nginx: `proxy_buffering off;`,
  увеличить `proxy_read_timeout`), иначе realtime «залипнет».
- HTTPS желательно; всё работает по одной ссылке `https://домен/`.
