# Кайфоград

Онлайн-приложение для командного квиза: команды заходят по одной ссылке, отвечают со своих устройств, счёт и таймер живут на сервере.

## Что есть

- Панель ведущей: ответы, таймер, счёт, пауза, следующий вопрос, завершение раунда, ручная правка баллов, итоги и финальное раскрытие.
- Экран команды: один раз выбрать слот, название и цвет, дальше только большой вопрос и ответ.
- Экран на проектор: только текущий вопрос для трансляции в зал.
- Вкладка Кайфоград / Miro: встроенная Miro-доска, ссылка на ручную карту и подсказка по ресурсам.
- Игровая логика с тестами: команды-слоты, приватность ответов, автоподсчёт, занятые цвета, ресурсы, финальный топ.

## Запуск

```bash
/Users/ekaterina/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node server.js
```

Открыть:

```text
http://localhost:4173
```

Для команд обычная ссылка:

```text
http://localhost:4173/
```

Они сами выбирают номер команды, вводят название и цвет. Номер команды после входа привязывается к этому устройству, чтобы чужая вкладка не могла отправить ответ за них.

Для тебя остаётся служебная вкладка:

```text
http://localhost:4173/?view=host
```

Код ведущей: `0306`. Опасные действия (`Сбросить игру`, `Финал 3-2-1`) требуют подтверждения.

## Сценарий игры

1. Команды заходят по общей ссылке, выбирают слот, название и цвет.
2. Ведущая открывает вкладку `Ведущая`, вводит код `0306` и нажимает `Начать игру`.
3. Вопросы внутри раунда идут автоматически по таймеру, но ведущая может поставить паузу, закрыть вопрос, перейти к следующему вопросу или завершить раунд вручную.
4. После последнего вопроса появляется `Раунд закончился`; ведущая нажимает `Завершить раунд`, чтобы показать итоги.
5. Следующий раунд начинается только по кнопке `Следующий раунд`.
6. После последнего раунда ведущая запускает финальное раскрытие `3-2-1`.

## Как приложение общается с телефонами (2 запроса)

Чтобы на конференц-вайфае с двумя десятками телефонов не было подлагиваний, данные разделены на два потока:

1. **Контент — один раз при загрузке страницы.** Тексты вопросов, варианты и картинки клиент забирает полным снимком (`/api/state`) один раз. Они не меняются по ходу игры.
2. **Живой стрим — только статусы.** Дальше сервер по SSE (`/api/events`) шлёт на каждое действие лёгкое обновление: какой сейчас вопрос, счёт, таймер, чей ответ принят. Тексты вопросов туда не попадают — клиент склеивает статусы с уже полученным контентом. Правильные ответы для показа на ревью/итогах приходят отдельным маленьким полем `reveals` (без «правильного» варианта — он остаётся только у ведущей).

За счёт этого обновление в стриме весит ~1.9 КБ вместо ~12 КБ, а суммарный трафик на 20 подключений в нагрузочной пробе упал с ~68 МБ до ~11 МБ.

## Защита

- Любая строка от участника (название команды, ответ) обрезается до 150 символов.
- Тело запроса ограничено 4 КБ — огромный запрос не «завесит» сервер на разборе (ответ `413`).

## Важная механика

- Команды можно тестировать без реальных названий: `Команда 1`, `Команда 2`, ...
- Команда позже вводит название и выбирает цвет.
- Занятый номер команды и занятый цвет нельзя использовать повторно.
- Команды не видят чужие ответы.
- Ведущая видит всё.
- Варианты, числа и текстовые ответы считаются автоматически.
- Таймеры: раунд 1 — 20 секунд, раунды 2–6 — 45 секунд, раунд 7 — 30 секунд, раунд 8 — 60 секунд на попытку.
- Раунд 8 — конкурс с шариками: ведущая запускает 60-секундную попытку для команды, может ставить паузу, завершать попытку и вручную начислять баллы; попытки можно запускать несколько раз.
- После раунда приложение подсказывает победителя, ресурс и цвет.
- Miro редактируют руками команды/админы во вкладке `Кайфоград / Miro`.
- Для локального и серверного теста ответы синхронизируются через общий Node.js-сервер и SSE-события.
